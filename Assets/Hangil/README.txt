# 제작 방식
- 아이작이 많은 작은 큐브로 이루어진 맵을 여러개 배치해놓은 것을 구현
- 큐브 위에 오브젝트를 추가한 것들을 하나의 세트로 설정
- 세트가 여러개 모여 하나의 맵으로 구성
- 맵을 여러개 만들어 다양한 맵들로 갈 수 있게 함.

# 맵 구성 요소
맵 구성 요소는 블럭과 오브젝트 2개로 구분
## 블럭
1. 바닥
- 문제 없이 지나가는 블럭
2. 장애물
- 아예 못 지나가는 블럭
ex) 벽, 장애물
3. 함정
- 지나갈 경우 사망 or 체력 감소 그런 쪽
ex) 용암바닥, 구멍, 적 추가 스폰 타입

## 오브젝트

1. 프롭
- 아무런 기능 없이 장식용으로 존재하는 것들
- 주로 맵 외곽부에 장식으로 넣을 예정
- 장애물 블럭에 프롭을 배치하여 이동이 불가능한 이유를 직관적으로 제공
 - 투명벽에 가려져서 못가는 문제 해결
2. 플레이어 스폰
- 해당 블럭 위에 배치하여 아군 스폰하는 위치를 설정
- 처음 게임 로딩할 때 스폰되는 위치 설정
3. 적 스폰
- 해당 블럭 위에 배치하여 적군이 스폰하는 위치를 설정
- 처음 게임 로딩할 때 일괄 스폰되게 함
의도 : 맵간 이동할 때 마다 다시 스폰되어 불필요한 반복 이슈를 해결
5. 로딩 포인트
- 해당 로딩 포인트에 플레이어가 닿았을 경우 다음 맵 로딩
- 적이 없을 경우 로딩 포인트 오픈
6. 보물상자
- 보물 상자는 아이템을 담고 있으며 맵 내의 열쇠를 획득해야 열림
7. 열쇠
- 열쇠는 같은 맵 내에 존재하며 보물상자와 1대 1로 연결
- 열쇠와 닿을 경우 보물상자의 잠금을 해체하는 방식
8. 히든 포인트
- 맵 내 숨겨진 위치로 직접 밟을 경우 발동
- 열쇠 or 아이템 등등 물품이 등장 가능.

# 세부 구현 사항

## 블럭
1. 바닥
- 바닥은 Cube 텍스쳐, 콜라이더를 사용
2. 장애물
- 장애물의 경우 큐브를 위에 한칸 올린 이후 렌더링을 투명하게 변경
3. 함정
- 함정의 경우 콜라이더를 큐브 전체로 하지 않고 큐브 밑부분만 부여
- 들어갈 수 있게 하며 콜라이더와 닿을 경우 특정 효과 발동

## 오브젝트
1. 플레이어 스폰
- 플레이어 스폰은 1회성으로 게임 시작할 때 중앙 맵에 1개만 존재
- 1회 사용되고 게임이 다시 시작될 때 까지 비활성화
2. 적 스폰
- 적 스폰도 1회성으로 각 맵의 소환 위치에 존재
- 중앙 맵에는 컨트롤을 숙달하기 위해 없음
- 게임이 시작할 때 1회 로딩되고 처치될 경우 다시 로딩되지 않음
3. 로딩 포인트
- 각 맵의 최소 1방향, 최대 4방향 존재하며 적이 없는 경우 오픈
- 처음 오픈될 경우 이후에 다시 맵에 진입할 때도 지속적으로 로딩
4. 보물상자
- 게임이 시작할 때 각 맵에 소환 위치에 존재
- 열쇠를 획득하지 않은 경우 상자 상태로 존재
- 열쇠를 획득한 경우 비활성화되고 그 위치에 아이템 스폰
- 아이템은 플레이어가 먹기 전까지 맵에 존재
5. 열쇠
- 열쇠는 보물상자가 있는 맵에 함께 존재
- 획득한 경우 비 활성화, 보물상자 잠금 해체
- 이후 다시 맵에 들어가더라도 비 활성화
